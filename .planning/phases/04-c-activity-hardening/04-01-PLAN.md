<objective>
Establish the CONC-02 ownership foundation: a `ThomazActivity` base class (between every activity and `brls::Activity`) that owns both the `alive` and `cancelled` lifetime guards and exposes a protected `runAsync(worker, onSync)` wrapper. The wrapper encapsulates the hand-rolled capture-by-value + `if (!alive->load()) return;` idiom that is currently repeated across 13 activities, making the use-after-free guard impossible to forget. The testable guard *decision* is extracted into a pure, Borealis-free `thomaz::core::run_if_alive` helper so the dropped-callback semantics (TEST-04b) can be host-tested under the C++17 `tests/` suite, which does not link Borealis.

Purpose: This is the locked-first piece (CONC-02 before CONC-03 before DEBT-03 — ROADMAP planning flags + RESEARCH §"Sequencing"). Plans 02/03 migrate activities onto this base; Plan 04 (CONC-03) builds curl cancellation on the base's `cancelled` flag. Nothing can migrate until the base and the pure helper exist.

Output: `source/app/thomaz_activity.hpp` (ThomazActivity + runAsync), `source/core/async_guard.hpp` (pure run_if_alive), `tests/test_async_guard.cpp` (TEST-04b).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| async pool thread → UI thread | A worker runs on the Borealis async pool while the owning activity may be destroyed; the `alive` guard mediates whether the UI continuation (`onSync`) is allowed to touch `this` |

## STRIDE Threat Register

This phase is an internal concurrency/robustness refactor with no auth/crypto/input-validation surface changes. There is no high-severity (network-exploitable) threat introduced by this plan. The relevant threat class IS the subject of the refactor: use-after-free / data races on the lifetime guard.

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering (memory) / use-after-free | `runAsync` continuation touching a destroyed activity | mitigate | `runAsync` captures `alive` as a `shared_ptr<atomic<bool>>` BY VALUE (the flag object outlives the activity via the worker's copy); the `brls::sync` continuation calls `run_if_alive(alive, onSync)`, which drops `onSync` when `!alive->load()`. The base destructor sets `*alive = false` so any in-flight continuation bails. |
| T-04-02 | Information disclosure / data race | concurrent read/write of the guard flag across pool and UI threads | accept→mitigate | The guard is `std::atomic<bool>` (lock-free load/store); no torn reads. No PII flows through this path. |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs in this phase — pure in-repo C++ refactor, zero new dependencies (RESEARCH §"Standard Stack": "Installation: None"). RESEARCH Package Legitimacy gate N/A. |

No high-severity threat exists in this plan; recorded explicitly per the security contract.
</threat_model>
