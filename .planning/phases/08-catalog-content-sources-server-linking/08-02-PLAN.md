<objective>
Build the device data layer that feeds the catalog: extend `IHttpClient`/`CurlHttpClient` with a
non-buffering streaming/Range `stream()` seam (designed here, consumed by Phase 10's install
engine), and add the `platform/games/` orchestration that fetches a Tinfoil index (capped,
TLS-fail-closed, redirect-safe), drives the pure recurse-plan bounds over nested `directories`,
persists an empty-by-default source list, and caches each source's index on SD.

Purpose: Content bytes must NEVER flow through the buffering `request()`/`HttpResponse.body`
path (OOM on multi-GB titles) — this plan establishes the streaming seam so Phase 10 builds on a
proven interface, and it wires the pure `core/games/` logic from Plan 01 to real network and SD
I/O. SRC-04's "empty by default" and SRC-02's custom-header-redirect security are enforced here.

Output: A streaming `stream()` added alongside the untouched `request()` path, plus
`index_fetcher`, `source_store`, and `catalog_cache` in a new `source/platform/games/` namespace.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| device → remote content server | index/content fetch over HTTP(S); redirects may target attacker hosts |
| device → SD card | source list + index cache written to plaintext-readable SD |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-04 | Information Disclosure | custom auth header leaked to a cross-host redirect (SRC-02) | mitigate | `index_fetcher` re-attaches the custom header only on a same-host redirect; `CURLOPT_UNRESTRICTED_AUTH` is never set; basic-auth-in-URL relies on libcurl's default cross-host `Authorization` strip (Pitfall 4). |
| T-08-05 | Tampering / Spoofing | MITM serving a forged index or downgraded TLS | mitigate | `apply_curl_tls` (fail-closed Verify) applied on both `request()` and `stream()` paths; no TLS relaxation introduced. |
| T-08-06 | DoS | pathological/huge index or deep/cyclic directories → console OOM | mitigate | 8 MiB index cap in `fetch_index` (Pitfall 2); `recurse_plan` bounds + visited-URL cycle tracking (Pitfall 3); `stream()` never buffers a whole body. |
| T-08-07 | Information Disclosure | credentials written to logs or to the SD list visibly | mitigate (SD plaintext accepted) | Keep credentials out of every `brls::Logger` line (redacted host only, Pitfall 7); SD plaintext storage is an accepted MVP limitation (E2E deferred), documented in source_store. |
</threat_model>
