<objective>
Build the catalog browse surface: a `CatalogActivity` that paints the cached index instantly,
refreshes in the background when online, renders one cover-art card per base title with a kind
chip, offers a grid<->list toggle plus live search/sort/content-filter, and a
`CatalogDetailActivity` that lists Base / Update v{n} / DLC rows with sizes and no install button.
All grouping/sort/filter/kind logic comes from `core/games/` (Plan 01); fetch/cache from Plan 02;
cover art and i18n from Plan 04 — the Activities only compose.

Purpose: CAT-01 (grid/list with cover art, name, size, reusing the titledb/icon UI) and CAT-02
(kind chips from the 64-bit title ID + filter/search). This is a direct structural clone of
`theme_browser_activity` + `theme_detail_activity`. No install affordance exists in Phase 8
(installs are Phase 10).

Output: `catalog.xml` + `catalog_detail.xml`, and the two Activities.
</objective>
