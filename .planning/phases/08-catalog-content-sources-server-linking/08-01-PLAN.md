<objective>
Build the pure, host-testable foundation of the catalog feature in a new `source/core/games/`
namespace: tolerant Tinfoil index parsing, 64-bit title-ID kind derivation + base grouping,
catalog sort/filter/search, bounded `directories` recursion planning, and the config-only
source-link sync codec. Every piece is decidable logic with no I/O, gated by the host doctest
suite.

Purpose: This layer is consumed by every later plan (platform fetch/store in 02, the cloud
sync route in 03, the cover-art service in 04, and the catalog/source UI in 05/06). Building
and proving it first under doctest means hardware/UI failures later are integration bugs, not
logic bugs (the milestone's host-tested-logic-first sequencing principle).

Output: Six new `core/games/*.{hpp,cpp}` units, six doctest files, a captured real-server
index fixture, and an updated `tests/Makefile` that compiles and runs them green.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| remote server JSON → device parser | Untrusted, community-format index JSON crosses into `parse_index` |
| device config → sync JSON | `source_link` codec produces JSON destined for the cloud API |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering / DoS | `parse_index` on malformed/hostile JSON | mitigate | `nlohmann::json::parse` with `allow_exceptions=false` + `is_discarded()` guard; skip bad entries, never hard-fail; emptiness is valid (no crash on adversarial input). Byte cap applied upstream in Plan 02. |
| T-08-02 | Information Disclosure | `serialize_source_link` leaking credentials into synced/catalog JSON | mitigate | Codec writes only label/url/authType; doctest asserts the credential value is absent from serialized output (Pitfall 7 / SYNC-02). |
| T-08-03 | DoS | unbounded `directories` recursion | mitigate | `recurse_plan` bounds (depth 3 / entries 50k / requests 256) are decided in pure core here; the fetch loop in Plan 02 enforces them + cycle tracking (Pitfall 3). |
</threat_model>
