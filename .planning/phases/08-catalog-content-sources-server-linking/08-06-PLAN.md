<objective>
Build the source-management entry surface: a `SourceListActivity` (whose header + Home wiring
landed in Plan 04) that starts EMPTY (SRC-04), lets the user add a remote source with optional
auth (SRC-02), exposes a local SD pseudo-source browsed through the same catalog surface (SRC-03),
one-tap syncs the config to the account (SYNC-01), and opens a picked source's catalog. Plus a
`local_source` scanner that synthesizes an index from SD `.nsp`/`.nsz` files so the local case
flows through the exact same `core/games/` grouping path.

Purpose: Closes SRC-02/03/04 and the SYNC-01 device wiring. Empty-by-default and credential
redaction are enforced at the UI; the local pseudo-source reuses the catalog/detail surface
verbatim (no separate browser).

Output: `source_list.xml`, `source_list_activity.cpp` (the .cpp for Plan 04's header), and
`local_source.{hpp,cpp}`.
</objective>
