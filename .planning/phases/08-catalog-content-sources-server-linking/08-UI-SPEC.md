## Spacing Scale

borealis XML uses fixed-pixel metrics on nodes (`padding*`, `margin*`, `cornerRadius`, `height`).
This phase adopts the scale already established across `home.xml` / `theme_browser.xml` /
`theme_detail.xml` / `game_list.xml`. All values are multiples of 4 except the two legacy
14/22 rhythm values the existing screens use â€” keep them ONLY to stay visually consistent with
the screens being cloned.

| Token | Value | Usage (this phase) |
|-------|-------|--------------------|
| xs | 4px | chip inner gap, label-to-sublabel (`marginBottom="4"`) |
| sm | 8px | inline gaps between filter chips (`marginRight="8"`), chip corner radius |
| md | 16px | default element spacing, card inner padding, detail-row gap |
| lg | 24px / 28px | screen `paddingBottom`, detail content top padding |
| xl | 32px | screen `paddingTop` (matches `game_list.xml`/`home.xml`) |
| 2xl | 40px / 48px | screen `paddingLeft`/`paddingRight` (40 in browser/detail, 48 in list/home) |

**Card/control metrics (locked to existing rhythm):**
- Cover-art grid card corner radius: **12px** (matches `detailPreview`); row/chip corner radius **8px**; primary action button corner radius **10px**.
- Filter/tab chip height: **40px** (matches `theme_browser` tabs); primary action button height: **56px** (matches `downloadButton`).
- Grid card cover image aspect: portrait box art (Switch eShop covers ~ 1:1.4); fixed card width per grid column modeling the `theme_browser` results layout.

Exceptions: legacy `14` and `22` spacing values (carried from the cloned screens for visual
parity â€” do NOT introduce new off-scale values beyond these).

## Typography

borealis has ONE bundled font family and does **not** expose multiple font weights â€” emphasis is
achieved by **font size + text color token**, not by weight. The "2 weights" rule from the web
template is satisfied here as **2 emphasis levels** (full-strength `text` color = "strong",
`text_dim` = "secondary"). Sizes below match the existing screens verbatim.

| Role | Size | Emphasis (color token) | Line Height | Used for |
|------|------|------------------------|-------------|----------|
| Display | 34px | strong (`thomaz/text`) | default | Home greeting only (existing) â€” NOT introduced by this phase |
| Heading | 24px | strong (`thomaz/text` / `#FFFFFF`) | 1.2 | Catalog detail title (`detailName`), source name in detail |
| Title | 17â€“18px | strong (`#FFFFFF` on tile / `thomaz/text`) | default | Home "Games" card label (17), primary action button label (18) |
| Body | 15px | secondary (`thomaz/text_dim`) | 1.45 | descriptions, source URL, sizes, sublabels, chip labels |
| Empty/Note | 18px (empty), 13px (note) | per state (see Color) | default | empty-state label (18, centered), action note (13, dim) |

Declared set for this phase: **24 / 18 / 15 / 13** (4 sizes) at **2 emphasis levels**
(strong `text`, secondary `text_dim`). The 34px display is pre-existing (home) and not used by
new Phase 8 screens. Line-height 1.2 for headings, 1.45 for wrapping body copy (matches
`detailName` / `detailDesc`).

## Color

Built entirely from the registered `thomaz/*` dark-theme tokens in `source/app/theme.cpp`.
Dark mode is forced. The 60/30/10 split maps to surface tokens + the violet accent ramp.

| Role | Token / Value | Usage |
|------|---------------|-------|
| Dominant (60%) | `thomaz/bg` `#0E0F14` + `thomaz/surface_0` `#14151A` | App backdrop + AppletFrame base behind catalog/source/detail screens |
| Secondary (30%) | `thomaz/surface_1` `#1A1C23` (cards/rows), `thomaz/surface_2` `#22242D` (focused/hover row + inactive tab/chip), `thomaz/surface_3` `#2B2E39` (active chips/track) | Cover-art cards, source-list rows, filter chips, detail panels |
| Accent (10%) | `thomaz/accent` `#7C5CFF` / `thomaz/accent_bright` `#9277FF` | Reserved list below â€” never "all interactive elements" |
| Positive | `thomaz/good` `#57C98A` | "Synced" confirmation state of the cloud-sync action ONLY |
| Destructive | `brls/warning` (borealis default red) via dialog button | Remove-source confirmation button ONLY |

**Module tile color (Home card):** the new "Games / Content" Home card uses a single distinct
hue per the per-module tile convention. No tile token currently maps to games â€” planner adds one
new `thomaz/tile_games` token (a hue not already used by cheats=violet / settings=teal /
saves=amber / mods=rose / backup=emerald; e.g. an indigo/blue) so the card reads as its own
module. Register it in `theme.cpp` alongside the existing `tile_*` colors.

**Accent (`#7C5CFF` / `#9277FF`) reserved for â€” EXPLICIT list:**
1. The current gamepad **focus highlight ring** (borealis `brls/accent`, applies on any focusable node â€” this is the platform focus affordance, not a per-element decoration).
2. The **active** tab/filter chip background (inactive chips use `surface_2` `#22242D`, matching the `theme_browser` `tabPacks` active / `tabThemes` inactive pattern).
3. The **primary action button** fill (Add source confirm, Sync to account, Refresh-when-primary) â€” modeled on `downloadButton` `accent_bright`.
4. The base/update/DLC **kind chip** uses tonal differentiation: Base = neutral `surface_3`; Update = accent `accent_soft`/`accent`; DLC = `good`-adjacent or neutral â€” exact mapping a planner detail, but accent stays reserved to â‰¤ these roles.

**Kind chip semantics (CAT-02):** Base / Update / DLC are distinguished by **text label +
background tone**, NOT by icon. Keep Base visually quietest (neutral surface), reserve the
brighter accent for Update/DLC so the eye finds "this title has extra content" fast.

**State colors:** offline/stale note (D-06) â†’ `thomaz/text_dim`; "catalog truncated (limit
reached)" note (D-07) â†’ `thomaz/text_dim`; error text â†’ `thomaz/text_dim` (matches
`detailError`); synced-OK â†’ `thomaz/good`.

## Interaction & Navigation Contract (borealis-specific)

This replaces the web template's pointer assumptions. All navigation is **D-pad / gamepad focus**.

| Concern | Contract |
|---------|----------|
| Entry | New focusable Home card (`gamesCard` / `contentCard`) in `home.xml`, wired in `home_activity.cpp`, pushes `SourceListActivity`. |
| Source list | Vertical focus list of source rows + a persistent "Add Source" affordance; **empty by default** (SRC-04) â†’ shows the empty state, focus lands on "Add Source". Local SD appears as a peer row. |
| Add-source flow | URL field + optional auth fields (basic-auth-in-URL / custom header / referrer) via borealis text-entry (swkbd); confirm = `Add Source`. |
| Catalog screen | Grid â‡„ compact-list toggle (D-03); search (name + title ID, D-04) via the existing `searchButton`â†’swkbd pattern; sort (name/size) + content-filter chips (has update / has DLC / base-only, D-04) as `surface_2`/accent chips. |
| Cache-first paint | On open, render cached SD index instantly (D-05), spinner only for the background refresh; if online always refresh (D-06), else show offline note. Never blank-flash: `loadingBox`/`emptyLabel`/`resultsBox` visibility dance from `theme_browser`. |
| Detail screen | One card per base title (D-01) â†’ detail lists `Base / Update v{n} / DLC` rows with sizes; `detailStatus` spinner/error gates the hidden `detailContent` (theme_detail pattern). NO install button (Phase 10). |
