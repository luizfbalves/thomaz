<objective>
Remove all client-side community-feed C++ code while preserving auth/session infrastructure. The key challenge is that IFeedClient (feed_client.hpp) is the type used throughout the activity chain for auth — it must be replaced with a clean IAuthClient interface before the feed files can be deleted.

Per D-08: delete feed_json.{cpp,hpp}, feed_types.hpp, http_feed_client.{cpp,hpp}, fake_feed_client.{cpp,hpp}, feed_client.hpp.
Per D-09: PRESERVE source/core/feed/session_codec.{cpp,hpp} and source/platform/feed/auth_store.{cpp,hpp} — these are auth/session infrastructure.

The removal sequence is:
1. Move Session and AuthResult from feed_types.hpp into session_codec.hpp (since session_codec is preserved and already includes feed_types.hpp; the types must survive).
2. Create source/platform/auth_client.hpp (IAuthClient), http_auth_client.{cpp,hpp}, fake_auth_client.{cpp,hpp} — same API as IFeedClient but named for auth, not feed.
3. Update all activities and main.cpp to use IAuthClient instead of IFeedClient.
4. Delete the 8 feed files (feed_json.*, feed_types.hpp, http_feed_client.*, fake_feed_client.*, feed_client.hpp).

CMakeLists.txt uses GLOB_RECURSE on source/*.cpp so no CMake edits are needed when files are added/deleted under source/.

Purpose: Client compiles cleanly with no dead references to IFeedClient, feed_client.hpp, or feed_types.hpp while login/register functionality continues to work via IAuthClient backed by HttpAuthClient.
Output: New auth_client platform interface; 8 feed files deleted; all activities updated; desktop build green.
</objective>
