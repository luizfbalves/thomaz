<objective>
Extract the TLS verification decision into a pure, host-compilable `tls_policy(bool ca_present) -> {verifypeer, verifyhost}` free function (D-06) in a new curl-free header `source/platform/tls_policy.hpp`, refactor `apply_curl_tls()` in `source/platform/curl_tls.hpp` to consume it on both branches, add a process-global `tls_insecure` latch set when the CA bundle probe fails, and add a host doctest (TEST-03) that exercises the `ca_present==false` fail-safe branch so a regression that silently disables verification fails CI.

The fail-safe NETWORKING behavior is unchanged (D-03 / SEC-03 lock): when the CA bundle is absent the app still degrades to no-verification. This plan only makes the decision logic host-testable and sets the flag the SEC-03 banner (Plan 03) will read.

Purpose: Create a clean test seam for the locked TLS fail-safe and a regression guard against accidental silent verification-disable.
Output: tls_policy.hpp; refactored curl_tls.hpp with insecure latch; test_tls_policy.cpp.
</objective>
