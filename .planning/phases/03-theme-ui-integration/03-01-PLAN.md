<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Filesystem → load_extract_state() | Untrusted JSON content from /switch/thomaz/config/.thomaz_extract.json |
| C++ code → i18n JSON files | Build-time only; files are in-tree, not user-supplied |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | load_extract_state() JSON parse | mitigate | nlohmann::json::parse with allow_exceptions=false; is_discarded() check before field access; j.value() with std::string() default — garbage/truncated file returns nullopt, never crashes |
| T-03-02 | Denial of Service | save_extract_state() write failure | accept | Write failure leaves old marker or no marker; load_extract_state() returns nullopt; status row shows "not extracted" — correct degraded behavior, no crash |
| T-03-03 | Tampering | sExtractionRunning bool | accept | Atomic flag prevents UI double-fire. Not a security boundary — no privileged state depends on its value; worst case is a missed guard, not a privilege escalation |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No package manager installs in this plan; all symbols are in-tree |
</threat_model>
