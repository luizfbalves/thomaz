Generalize the single-target extraction spike into the full multi-title extraction engine.

qlaunch layouts (including common.szs per D-01a), Psl.szs, and MyPage.szs in /themes/systemData/.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| NCA RomFS → szs_buf | Decrypted bytes from hactool fork; not yet validated; must pass is_structurally_valid_szs before write |
| szs_buf → SD card write | Structural validation (D-04) gates this; a corrupt extraction is rejected instead of written |
| romfs_key basename → out_path | Path construction: basename derived from `/lyt/foo.szs` → `foo.szs`; flat layout prevents traversal; no `..` possible in Nintendo RomFS paths |
| /lyt/ filter scope | All `/lyt/*.szs` from three titles; Nintendo firmware NCAs only contain layout files in /lyt/; not user-supplied data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03 | Tampering | nca_romfs_filter (widened prefix) | mitigate | Prefix `/lyt/` scopes capture to the layout directory only; no other RomFS paths are extracted; NCA paths come from lr resolver, not user input |
| T-02-04 | Tampering / Integrity | is_structurally_valid_szs | mitigate | Full Yaz0-decompress + SARC-unpack (D-04) rejects corrupt or partially-decrypted buffers before SD write; catches bad decrypt silently (hactool crash scenario) |
| T-02-05 | Elevation of Privilege | out_path derivation (romfs_key → szs_name) | mitigate | szs_name is the basename (last '/' to end) of a Nintendo RomFS path; Nintendo RomFS cannot contain `..` components; output is always base_layout_dir() + "/" + szs_name (flat, no user influence on path) |
| T-02-06 | Denial of Service | close_privileged_session leak on abort | mitigate | close_privileged_session() called on every exit path: D-02a systemic abort AND normal post-loop exit; prevents BIS/SPL session leak on failure |
| T-02-07 | Info Disclosure | derived header_key in scope during title loop | mitigate | Header key lives in KeyDerivationOutput.header_key (vector); wiped inside nca_extract_switch.cpp after nca_free_section_contexts (existing T-01-10 mitigation); no logging of key value |
| T-02-08 | Tampering | overwrite-in-place (D-03) | accept | D-03 is an explicit user decision: re-extract overwrites stale szs. The structural validation (T-02-04) ensures only a valid new szs replaces the old one |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs in this plan — pure C++ source modifications only |
</threat_model>
