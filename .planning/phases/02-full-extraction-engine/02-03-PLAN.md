<objective>
Implement the Phase 2 extraction engine by composing the proven Phase 1 primitives
into a multi-title, best-effort driver, and widening the one seam that gates it.

Two tightly-coupled changes (same data path, must ship together):
1. Widen `nca_romfs_filter` in nca_extract_switch.cpp from exact-name match to a
   `/lyt/` directory-prefix match (D-01), keeping exact single-name matching for
   the existing single-target caller. This is what lets the driver capture every
   `/lyt/*.szs` (incl. `common`, Psl, MyPage) in one NCA pass.
2. Add `extract_all_base_layouts()` to firmware_extract_switch.cpp: applet gate
   first (TAKEOVER-01); firmware-version log once; open the privileged session and
   derive the SPL header key ONCE (D-02a); loop titles 0100000000001000 (qlaunch),
   0100000000001007 (Psl), 0100000000001013 (MyPage), each resolved via
   resolve_nca_path and extracted with the /lyt/ prefix filter and the reused key;
   per captured file, validate with is_structurally_valid_szs (D-04) and write the
   RomFS basename flat into base_layout_dir() with trunc overwrite (D-03); collect
   per-part failures and continue (D-02); close the session exactly once on every
   exit path.

The single-target extract_base_layout() and its live caller (theme_detail_activity.cpp)
are KEPT unchanged — the new driver is additive.

Purpose: deliver EXTRACT-01/02/03 — all eight qlaunch parts plus Psl and MyPage
extracted to the flat /themes/systemData/ layout in one run.
Output: widened RomFS filter + extract_all_base_layouts() driver.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| applet mode → privileged FS/SPL | BIS/SPL access requires Application (title-takeover) mode; the applet gate is the access-control boundary (TAKEOVER-01 / V4). |
| firmware NCA (untrusted decrypt) → SD theme dir | A mis-decrypted/corrupt buffer must not reach the apply path or be written as a valid base (V5 / integrity). |
| RomFS key (title-controlled string) → SD output path | The output filename is derived from the RomFS key basename; it must land only inside base_layout_dir() (path-traversal / EoP). |
| privileged session handle (g_session) | Must be opened once and torn down on every exit path; a leaked handle is a resource leak holding privileged FS open. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Elevation of Privilege | extract_all_base_layouts() in applet mode | mitigate | Applet gate is the FIRST statement; returns the relaunch message with ok=false before any setsys/SPL/BIS init (Task 2 acceptance: gate precedes session open). |
| T-02-08 | Tampering / Integrity | corrupt szs written to /themes/systemData | mitigate | Every captured buffer passes is_structurally_valid_szs (D-04 Yaz0+SARC) before write; invalid buffers go to failed_parts, never overwrite a good file. |
| T-02-09 | Elevation of Privilege | output path traversal via RomFS basename | mitigate | Output is always base_layout_dir() + "/" + basename(romfs_key); only the last path segment is used (substr after final '/'), no caller-supplied directory; write_file/ensure_parent_dirs target only the SD theme dir. RomFS keys are firmware-controlled "/lyt/NAME.szs" with no ".." segment after basename extraction. |
| T-02-10 | Denial of Service / resource leak | privileged session not closed | mitigate | close_privileged_session() called on the key-derive abort path and exactly once after the loop; never inside the loop (Pitfall 2). Acceptance verifies no in-loop close. |
| T-02-11 | Availability | partial run leaves user with no base | mitigate | D-03 overwrite-in-place per validated part (not wipe-then-extract); best-effort means a missing optional title still leaves qlaunch parts written. ok=false never co-occurs with non-empty written_parts. |
| T-02-12 | Information Disclosure | header key leakage via driver | accept | Key handling is owned by key_loader/nca_extract (Phase 1: key wiped after use, only all-zero flag logged). The driver only passes kdo.header_key by reference into extract_szs_from_nca; it never logs or stores the key. No new disclosure surface. |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this plan; only in-tree Phase 1 primitives + SarcLib (already vendored) are composed. RESEARCH.md Package Legitimacy Audit = not applicable. |
