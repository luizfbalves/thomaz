# Requirements: thomaz — v1.1 Switch-Only Simplification

**Milestone goal:** Remove the desktop (PC/SDL2/GLFW) build target entirely so the source tree targets only Nintendo Switch, reducing maintenance surface — with **zero change to the shipped `.nro`**, proven by a green host doctest suite (`tests/Makefile`) and a clean Switch build (`scripts/build-switch.sh`).

**Type:** Internal refactor / removal (no new user-facing features).

---

## v1.1 Requirements

### Build System (BUILD)

- [ ] **BUILD-01**: `CMakeLists.txt` has no `PLATFORM_DESKTOP` configuration path — the tree configures and builds only for the Switch toolchain (devkitPro); the `elseif (PLATFORM_DESKTOP)` link branch and the `PLATFORM_DESKTOP` packaging branch are gone, along with dual-target comments.
- [ ] **BUILD-02**: The desktop-only helper scripts `scripts/build-desktop.sh` and `scripts/run-desktop.sh` are removed from the repo.
- [ ] **BUILD-03**: A clean Switch build via `scripts/build-switch.sh` (devkitPro Docker image) succeeds end-to-end after all removals, producing `build_switch/thomaz.nro`.

### Source Simplification (SIMPL)

- [x] **SIMPL-01**: The five desktop platform stub pairs are deleted from `source/platform/`: `save_service_fake.{cpp,hpp}`, `title_service_fake.{cpp,hpp}`, `fake_auth_client.{cpp,hpp}`, `themes/firmware_extract_fake.cpp`, and `sysmod/sysmod_store_fake.{cpp,hpp}`.
- [x] **SIMPL-02**: Every platform-selection seam that chose a `*_fake` vs `*_switch` *implementation* now unconditionally uses the Switch implementation (these were all in `main.cpp`/`home_activity.cpp`), and each affected `*_switch` impl is the sole implementation behind its interface. _Scope clarification (Option D, 2026-06-05):_ platform-portability `#ifdef __SWITCH__ … #else … #endif` seams that return a platform path string or a host-compilable fallback impl (and `_WIN32`/`localtime_r` seams) are **retained** — they are not desktop *stub-selection* and are required to keep the host doctest suite green.
- [x] **SIMPL-03**: No `source/` file references a deleted desktop stub or a desktop-only symbol/include (SDL2/GLFW/`PLATFORM_DESKTOP`-guarded code); a repo grep for desktop-only constructs in `source/` returns nothing.

### Verification & Documentation (VERIF / DOC)

- [ ] **VERIF-01**: The host doctest suite (`tests/Makefile`, `g++`) builds and passes unchanged after removal, confirming core and platform-neutral logic is unregressed; the retained test double `saves/fake_cloud_save_client.*` is still compiled by the suite.
- [ ] **DOC-01**: Repository docs and code comments no longer instruct or imply a desktop PC build — the `CMakeLists.txt` header comment, README/build docs, and any `AGENTS.md`/`CLAUDE.md` build notes are updated to describe a Switch-only tree with the host doctest + Switch-build verification flow.

## Out of Scope

- **Re-adding or preserving any desktop/PC build capability** — the explicit goal is full removal; PC GUI iteration and the non-devkitPro full-tree compile check are knowingly given up.
- **Removing `saves/fake_cloud_save_client.*`** — it is a host-doctest test double, not a desktop GUI stub; it stays.
- **Changing any runtime behavior of the Switch `.nro`** — this is a build/source removal only; the shipped app must behave identically.
- **Touching the cloud API (`api/`)** — unaffected by desktop removal.
- **Reworking the `core/` vs `platform/` clean-architecture split** beyond collapsing the now-single-target selection seams.
