<objective>
Complete CONC-03 cancellation scope (D-03): wire the `cancelled` flag into the FIVE remaining activity-owned network transfer sites that 04-04 did not cover — themes (theme_download + theme_detail + theme_browser) and browse GETs (mod_browser + cheat_detail + game_list). The transport plumbing (XFERINFOFUNCTION abort hook in http_client_curl.cpp; optional cancelled param in download_file; cancelledFlag() accessor on ThomazActivity) was completed in 04-04. Every unwired site is a one-line omission: capture `auto cancelled = this->cancelledFlag();` before each `runAsync`, then set `req.cancelled = cancelled` on each HttpRequest, or pass `cancelled` into `download_theme`.

Purpose: D-03 mandates ALL activity-owned network transfers cancel on activity destruction. With theme/browse GETs unset, destroying those activities leaves in-flight pool threads running to completion — the resource leak CONC-03 targets. After this plan, D-03 is fully satisfied.

Output: seven files modified (theme_download.hpp/cpp + five activity .cpp files); desktop build clean; host tests still pass.
</objective>
