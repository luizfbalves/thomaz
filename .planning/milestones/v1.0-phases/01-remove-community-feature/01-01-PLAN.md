Remove all community-feature code from the API layer: delete the posts, feed, and users route files; strip @fastify/multipart and @fastify/static plugin registrations from app.ts; remove community-only serializers and the feed-page lib helper; and scrub the test suite of any test that references community endpoints or models.

The Prisma schema edit and migration happen in Plan 03. This plan only touches TypeScript and test files.

Purpose: Clear the server-side community surface area so the Vitest suite runs green without dead references to Post/Like/Comment or the removed routes.
Output: app.ts registering only auth + saves routes; serializers.ts carrying only save-slot and user-dto helpers; a clean Vitest run (noting that the migration has not run yet — Prisma client types regenerate from schema so type-check passes; migration is BLOCKING in Plan 03).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet → /uploads/* | Previously: @fastify/static served any file under UPLOAD_DIR publicly — this was the SEC-01 exposure. After this plan: the static route is gone; no file under uploads/ is publicly reachable. |
| Authenticated client → /saves/* | Save blobs are accessible only via the authenticated saves routes, not via a static path. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | @fastify/static / uploads/saves/ | mitigate | Removing @fastify/static registration closes the public static path entirely — no save blob is reachable without a valid auth token (via /saves/:titleId?includeData=1). This IS the SEC-01 root-cause fix per D-04 and D-10. |
| T-01-02 | Tampering | @fastify/multipart image upload | mitigate | Removing @fastify/multipart eliminates the MIME-spoof upload path and unbounded file upload surface; VAL-01 and VAL-02 are rendered moot (Out of Scope in REQUIREMENTS.md). |
| T-01-03 | Elevation of Privilege | /users/:username public profile endpoint | mitigate | Deleting users.ts removes the unauthenticated public profile endpoint and its fetchFeedPage call. |
| T-01-04 | Information Disclosure | api/src/lib/serializers.ts Post/imageKey exposure | mitigate | Removing toPostDto eliminates any remaining imageUrl construction using PUBLIC_BASE_URL/uploads/ path. |
| T-01-SC | Tampering | npm installs (no new packages in this plan) | accept | No new npm packages installed; existing packages already audited in STACK.md. |

**Residual check:** Removing @fastify/static does not affect any auth-protected route. The saves routes (`/saves`, `/saves/:titleId`) are Express-style Fastify route handlers with `preHandler: [app.authenticate]` — they are not static file handlers and are unaffected by this removal. ensureUploadDir still runs on startup to ensure the saves blob directory exists for writeSaveBlob in saves.ts.
