<objective>
Lay the database and logging foundation for the security phase: add the `RevokedToken` Prisma model with a hand-numbered migration, apply that migration to the test Postgres, and replace `Fastify({ logger: false })` with an `envToLogger` map that enables redacted pino logging in production/development while keeping the test suite silent.

Output: `RevokedToken` model + applied migration, regenerated Prisma client (typed `prisma.revokedToken` delegate), `envToLogger` map in `app.ts`, `pino-pretty` devDependency.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client ‚Üí API request headers | Untrusted bearer tokens and cookies arrive in `req.headers`; logging must not echo them |
| API ‚Üí Postgres (migrations) | Schema change applied via non-interactive `migrate deploy` on the deploy host |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | Request logging (`app.ts` logger) | mitigate | pino `redact` removes `req.headers.authorization` and `req.headers.cookie`; production branch is an object carrying `redact` (not bare `true`); bodies are not serialized by Fastify's default request serializer (D-11) |
| T-02-02 | Tampering | Prisma migration history | mitigate | Use hand-numbered `migrate dev` authoring + `migrate deploy` application (never `db push`), mirroring the existing `20250603150000_refresh_tokens` migration; preserves migration-history integrity |
| T-02-03 | Denial of Service | Missing migration on test/prod DB | accept‚Üímitigate | Migration-applied precondition is an explicit acceptance criterion (`migrate status` clean) so the SEC-02 suite cannot silently pass against a schema-less DB |
| T-02-SC | Tampering | npm install of `pino-pretty` | mitigate | `pino-pretty` is `[VERIFIED]` in 02-RESEARCH.md Package Legitimacy Audit (published by pinojs, cited by official Fastify Logging docs, 8-yr registry history); single canonical package, no blocking checkpoint required |
