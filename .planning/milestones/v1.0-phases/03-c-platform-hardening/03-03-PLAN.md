<objective>
Deliver the SEC-03 persistent on-screen TLS-insecure warning (D-01, D-02, D-02a). Because there is no single brls::Application shell (each activity builds its own AppletFrame — RESEARCH), the app-wide banner is realized via a SHARED HELPER install_tls_warning_banner(brls::Activity*) mirroring the existing install_header_username precedent (app_header.cpp:10-27), called from each activity's onContentAvailable alongside the username helper, gated on the process-global tls_insecure latch (set by Plan 02 when ca_ok==false).

This is NOT a custom overlay and NOT log-only (D-01 rejects both). Fail-safe networking behavior is UNCHANGED (D-03) — the banner is additive and non-blocking.

Purpose: Make the silent TLS downgrade visible across every screen while the insecure state persists.
Output: tls_banner.{hpp,cpp}; banner wiring in 14 activities; i18n key in 5 locales.
</objective>
