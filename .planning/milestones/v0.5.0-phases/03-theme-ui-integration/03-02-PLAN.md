<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| UI thread → brls::async worker | ExtractAllResult crosses from worker thread into UI thread via brls::sync lambda capture |
| alive guard → activity lifetime | Activity may be destroyed while async is in flight |
| sExtractionRunning → both entry points | Global atomic prevents concurrent extraction from browser + detail page |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Denial of Service | Double-trigger from both entry points (Pitfall 2) | mitigate | sExtractionRunning std::atomic_bool: exchange(true) in both doExtract() implementations; reset false on all brls::sync exit paths including alive=false guard |
| T-03-05 | Denial of Service | spinnerDialog dangling after activity pop | mitigate | spinnerDialog->dismiss() is called BEFORE the alive->load() check in brls::sync — dialog always cleaned up regardless of activity state (Pitfall 5) |
| T-03-06 | Information Disclosure | systemic_error string shown in results dialog | accept | systemic_error comes from thomaz-owned engine code (applet mode message, key derivation message) — no filesystem paths, no cryptographic material, no Nintendo SDK error codes that would be sensitive |
| T-03-07 | Tampering | Concurrent C++ on non-atomic busy field | mitigate | busy field only written/read on UI thread (doExtract guards at top run on UI thread; brls::sync guarantees UI thread for reset); no data race |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No package manager installs in this plan |
</threat_model>
