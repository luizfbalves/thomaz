<objective>
Port the privileged key-resolution chain from exelix `key_loader.cpp` (the `__SWITCH__` branch) into
thomaz: initialise pmdmnt/spl/splCrypto, mount the raw BIS System partition, resolve the qlaunch
title's NCA path through the Location Resolver (`lr`), and derive the NCA header key on-device via SPL
from PUBLIC key sources — no `prod.keys` (EXTRACT-04). This is real, production-shaped keeper code
(D-01) that plan 04 composes with the hactool wrapper (plan 03), and that Phase 2 extends for more
titles.

Purpose: EXTRACT-04 — extraction succeeds WITHOUT the user supplying a `prod.keys` file because the
per-console header key is derived on the device through SPL hardware key slots. The pinned PUBLIC key
sources (header_kek_source / header_key_source) come from a recorded Atmosphère release (D-07); SPL
turns them into the per-console key. No secret is embedded, no derived key is logged or written to disk.

Output: `key_loader_switch.{hpp,cpp}` — a platform-neutral interface (no libnx types in the header)
plus the `__SWITCH__`-guarded real implementation. This plan does NOT add the entry point or applet
gate (plan 04) and does NOT call hactool (plan 03).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| applet/Application → privileged libnx services | BIS/SPL/lr/pmdmnt require Application-mode permissions; without them these calls fail. (Gate itself is plan 04; this file must still check every Result and fail-closed.) |
| derived key material → process memory | The 32-byte header key is sensitive; it must stay in memory only, never logged or persisted. |
| firmware NCA path (lr) → file open | A resolved path is untrusted input crossing into a privileged file open. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | Information Disclosure | Derived NCA header key | mitigate | Header key derived on-device via SPL, held in-memory only; NEVER logged, never written to disk, never embedded as a secret (only PUBLIC sources are compiled in). EXTRACT-04 keyless guarantee. |
| T-01-05 | Tampering / Integrity | Unchecked libnx `Result` | mitigate | Check `R_SUCCEEDED`/`R_FAILED` on every pmdmnt/spl/splCrypto/fs/lr call; surface a human-readable error; unmount/exit on every failure path (ASVS V5). |
| T-01-06 | Elevation of Privilege | Raw BIS System partition | mitigate | Mount System partition READ-ONLY in intent — read NCAs only; never write to BIS. The only write target is the SD card (plan 04). |
| T-01-07 | Spoofing / Input Validation | lr-resolved NCA path | mitigate | Validate the resolved path form and rewrite exactly per reference; log the raw path so a firmware drift is visible (Pitfall 5) before opening the file (ASVS V5). |
| T-01-08 | Information Disclosure / Legal | Embedded key sources | accept | Only PUBLIC Atmosphère key SOURCES are compiled in (not decrypted keys); pinned + attributed (D-07). Low risk — these are public values. |
</threat_model>
