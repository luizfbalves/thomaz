<objective>
Re-vendor the two dependencies Phase B excluded — the exelix hactool fork and a CMAC-enabled
mbedtls — and wire both into the CMake build as PLATFORM_SWITCH-only static libraries, then
record their provenance. This reverses the Phase B exclusion (D-06) and is the build foundation
every later Phase 1 plan compiles against. Nothing in plans 02-04 links without this.

Purpose: this plan is the build precondition for EXTRACT-04 (keyless on-device decryption). hactool
needs AES-CMAC (`MBEDTLS_CMAC_C`), which the devkitPro `switch-mbedtls` portlib (v2.28.10) is built
WITHOUT. So a second, CMAC-enabled mbedtls must be built from source (D-06) and must coexist on the
static link line with the portlib mbedtls that curl uses (Pitfall 1) — without it the SPL-derived-key
decrypt path (EXTRACT-04, implemented in plans 02-04) cannot link. This plan contains NO privileged
extraction logic — it is purely vendoring + build wiring + attribution.

Output: `lib/hactool/` (vendored fork + README + LICENSE), `lib/mbedtls/` (pinned source + CMAC
build config + README), modified `CMakeLists.txt`, and updated `THIRD_PARTY.md`.
</objective>
