<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| applet ↔ Application mode | The gate decides whether privileged calls are attempted at all; must fail-closed in applet mode. |
| decrypted szs buffer → SD write | Untrusted decrypt output crosses into a file write on the user's SD; validate before writing. |
| extraction output → downstream consumer | The written file is what `base_present_for` / "Aplicar Tema" trusts; a garbage file would silently break apply. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-14 | Denial of Service (silent) | Applet-mode privileged call | mitigate | `appletGetAppletType() != AppletType_Application` gate runs FIRST, before any service init / `fsOpenBisFileSystem`; returns a clear relaunch message and exits cleanly (TAKEOVER-01 / Pitfall 3). |
| T-01-15 | Tampering / Integrity | Garbage szs written on bad decrypt | mitigate | Validate the buffer is non-empty + has valid szs magic (SARC/`Yaz0`) BEFORE writing; never overwrite a good file with empty/garbage bytes (Pitfall 2). |
| T-01-16 | Elevation of Privilege | Write to wrong/privileged path | mitigate | Write ONLY to `cfw_paths::base_szs_path` on the SD (flat `/themes/systemData/`); never write to the BIS System partition (read-only intent); never the exelix subdir (Pitfall 6). |
| T-01-17 | Information Disclosure / Copyright | Extracted Nintendo szs | mitigate | The szs is written ONLY to the user's local SD; never uploaded/distributed (the milestone explicitly excludes distributing extracted assets). |
| T-01-18 | Tampering | Unchecked Result on the compose path | mitigate | Propagate every error string from key_loader/nca_extract; tear down services on every exit path; return a human-readable failure (ASVS V5). |
</threat_model>
